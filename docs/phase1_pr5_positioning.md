# Phase 1 · PR5 · Posicionamiento de producto

Este documento cierra el bloque **1.5 Posicionamiento de producto** del roadmap.

## Statement de posicionamiento

**openMiura es una governed agent operations platform para organizaciones que necesitan agentes auditables, controlados y extensibles, con despliegue serio y gobierno operativo.**

No compite como asistente personal local-first orientado a consumo; compite como plataforma para operar agentes en contexto B2B y entornos controlados.

## ICP

- equipos de ingeniería interna
- IT corporativo
- operaciones y soporte
- compliance / risk / governance
- empresas que quieren agentes con tools, memoria y trazabilidad
- despliegues on-prem, híbridos o de red controlada

## Casos de uso prioritarios

- broker unificado para agentes internos
- copilotos operativos con tools aprobadas
- workflows auditables con aprobaciones humanas
- integración multicanal bajo permisos y políticas
- operación de agentes por tenant/workspace en fase enterprise

## Diferenciadores clave

- broker HTTP como punto central de integración
- permisos y políticas explícitas
- trazabilidad y auditoría
- arquitectura por capas y contratos estables de extensibilidad
- orientación a operación, no solo a conversación

## Mensajes que no debemos usar

- “otro clon de asistente personal”
- “solo un wrapper de LLM”
- “chatbot multicanal” como mensaje principal
- claims de autonomía total sin control humano

## Comparativa corta frente a OpenClaw

### openMiura gana en
- gobierno del agente
- integración B2B
- extensibilidad contractual
- auditoría y observabilidad
- camino enterprise claro

### OpenClaw sigue ganando en
- experiencia más consumer
- visibilidad de ecosistema
- companion experience y amplitud de canal orientada a usuario final

## Slogan de trabajo

**Operate AI agents with control.**

Alternativas:
- **Governed agents for real operations**
- **The control plane for enterprise agents**

## Narrativa tipo landing B2B

### Headline
Controla, audita y opera agentes de IA con una plataforma pensada para organizaciones.

### Subheadline
openMiura unifica broker, herramientas, memoria, políticas y observabilidad para desplegar agentes útiles sin perder control operativo.

### Bloques clave
1. **Integración** — un broker HTTP y multicanal coherente
2. **Gobierno** — permisos, políticas y aprobaciones
3. **Ejecución** — tools, memoria y workflows listos para operación real
4. **Extensibilidad** — SDK público y contratos estables
5. **Enterprise path** — base preparada para tenant, workspace, SSO y compliance
