# Publication reuse note

Use the Sprint 5 publication pack like this:

- **Medium article:** `docs/media/medium_article_final.md`
- **Medium metadata and screenshot captions:** `docs/media/medium_article_publication_pack.md`
- **Stable GitHub Release body and short announcement:** `docs/release/stable_release_final_pack.md`
- **Canonical proof point:** `docs/demos/canonical_demo.md` and `docs/walkthroughs/canonical_runtime_governance_walkthrough.md`
- **Screenshot sequence:** `docs/media/screenshot_plan.md`

Recommended publication order:

1. cut `v1.0.0` and attach the official stable assets;
2. publish the GitHub Release using the stable release final pack;
3. capture screenshots using the screenshot plan;
4. publish the Medium article using the final article and publication pack;
5. reuse the same narrative in any short announcement or README release section.
